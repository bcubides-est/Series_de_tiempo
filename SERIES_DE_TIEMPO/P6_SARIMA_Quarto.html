10.1 Ejercicio de SimulaciÃ³n SARIMA

En esta prÃ¡ctica simulamos distintos procesos estacionales para identificar componentes SARIMA.
Recordemos el operador de diferencia estacional de periodo (s): \[ \Delta_s = 1 - B^s, \text{y el modelo multiplicativo} \\ SARIMA((p,d,q),(P,D,Q)[s]): \\ \Phi(B^s),\phi(B),(1 - B)^d,(1 - B^s)^D X_t ;=;\Theta(B^s),\theta(B),w_t. \]

10.2 1. SAR(1) estacional ((0,0,0)(1,0,0)[12])

Simulamos \[X_t = \Phi*1,X*{t-12} + w_t,\quad \Phi\_1=0.9,\] y comparamos su ACF/PACF teÃ³rica y muestral.

Code 
set.seed(666)
phi <- c(rep(0,11), .9)
sAR <- ts(arima.sim(list(order=c(12,0,0), ar=phi), n=37), frequency=12)

layout(matrix(c(1,1,2,1,1,3), nc=2))
par(mar=c(3,3,2,1), mgp=c(1.6,.6,0))

# Serie con marcadores para cada mes
plot(sAR, axes=FALSE, main="SAR(1) estacional", xlab="AÃ±o", type="o")
months <- c("J","F","M","A","M","J","J","A","S","O","N","D")
points(sAR, pch=months, col=rainbow(12), cex=1.2)
axis(1, 1:4); abline(v=1:4, lty=2, col=gray(.7)); axis(2); box()

# ACF y PACF teÃ³rica
ACF  <- ARMAacf(ar=phi, ma=0, 100)
PACF <- ARMAacf(ar=phi, ma=0, 100, pacf=TRUE)

plot(ACF,  type="h", xlab="Rezago", main="ACF teÃ³rica",  ylim=c(-.1,1)); abline(h=0)
plot(PACF, type="h", xlab="Rezago", main="PACF teÃ³rica", ylim=c(-.1,1)); abline(h=0)

InterpretaciÃ³n:
- La ACF exhibe picos cada 12 lags, caracterÃ­stica de estacionalidad de periodo (s=12).
- La PACF muestra un solo pico en rezago 12, indicando un tÃ©rmino SAR(1).

10.6.1 Notas de uso y modelado

  1. IdentificaciÃ³n:
    • ACF estacional: picos en mÃºltiplos de (s=12).
    • PACF estacional: corte despuÃ©s de (P) en rezago 12.
  2. DiferenciaciÃ³n:
    • Use nsdiffs para decidir el nÃºmero de diferencias estacionales (D).
    • Use ndiffs para diferencias regulares (d).
  3. SelecciÃ³n de modelo:
    • Combine patrones ACF/PACF con criterios AIC/BIC.
    • Estime con auto.arima(..., seasonal=TRUE) o sarima.
  4. DiagnÃ³stico:
    • Verifique residuos (ACF, Ljungâ€“Box).
    • Compruebe estacionariedad tras diferencias regulares y estacionales.
  5. PronÃ³stico:
    • Una vez identificado y diagnosticado, use el modelo para pronÃ³sticos con forecast.

Este flujo corresponde al mÃ©todo Boxâ€“Jenkins extendido a modelos estacionales SARIMA.
